Same Structure, Every Entity
Every white label entity ‚Äî whether an L5 solo operator in Davao or OFW Bank with 500 branches ‚Äî has the identical 7-role structure. Same schema, same portal, same rules. Only the scale and custom role names differ.
üîë
CircleFi Creates One Account
For every white label entity, CircleFi Super Admin creates only the Entity Owner account. Every other role ‚Äî Admins, Supervisors, Finance, Support, Tech, Managers ‚Äî is invited and managed by the partner independently.
üëÅÔ∏è
CircleFi Always Has Oversight
Super Admin and Ops Admin have read-only access to all entities at all times. Non-negotiable. Stated in the Franchise Agreement. Partners cannot remove this access. CircleFi can freeze, take over circles, or revoke Entity Owner in a breach.
‚úèÔ∏è
Custom Role Names
System role permissions are fixed and cannot be changed. But the display name is customisable ‚Äî RCBC calls their Supervisor a "Branch Circle Coordinator," OFW Bank calls Finance their "Treasury Officer." The permissions underneath never change.
üîí
Data Isolation via RLS
Every entity user's JWT contains their entity_id and branch_id. Supabase Row Level Security enforces isolation at the database layer. A partner with a raw API key physically cannot query another entity's data.
üìã
7-Year Audit Trail
Every write action is logged: actor, role, entity, action, before/after values, IP, timestamp. Logs are append-only, never deleted. Exported monthly as tamper-evident signed PDFs. Retained 7 years per BSP compliance requirement.
